# 우아한 테크코스 프리코스 - 오픈 미션: 프리코스 챌린지

## 미션 설명
### 진행 방식
1. 프리코스 경험 기반으로 관련된 분야의 미션을 설계하고 구현한다.
2. 미션 주제는 자유롭게 정한다.
3. 스스로 도전하고 싶은 목표를 설정하고, 그에 맞는 실행 계획을 세워 미션을 진행한다.
4. 2주 동안 자료 조사, 기획, 실행 후 결과물을 제출한다.

### 미션 예시
```
낯선 도구 해커톤
설명: 난이도는 평범하나 평소에 잘 사용하지 않는 개발 도구나 언어(예: Rust, Kotlin, WebAssembly, 하드웨어 보드 등)로 문제를 해결한다.
과제: 반드시 동작하는 작은 결과물(예: 간단한 앱, CLI 툴, 시뮬레이션) 제출한다.
```

```
고난도 문제 해커톤
설명: 평소에 익숙한 기술을 사용하여 난이도가 높은 문제를 해결한다.
과제: 난도가 높은, 정답이 없는 문제를 정의하고, 해결해 가는 과정, 결과물을 제출한다.
```

```
제한 협업 미션
설명: 2~3명이 한 팀이 되어 처음 보는 사람과 협업한다.
과제: 제한 조건을 만들어서 진행한다.
예) “소통은 하루 15분 화상회의만 가능”, “코드 작성은 깃허브 이슈/PR로만”
```

## 프로젝트 명: 오늘의 맛기온
### 💡기획 의도
* **문제점**: 많은 사람들이 음식을 주문할 때 겪는 가장 큰 어려움은 `음식 선택에 대한 많은 피로`입니다. 배달 앱에서 음식을 찾는 시간을 많이 소요하고 있습니다.
* **해결**: 문제점을 해소하기 위해 `기온 기반 맞춤 AI 음식 추천 서비스`를 기획했습니다.

객관적으로 현재의 날씨, 온도, 체감온도, 미세먼지 등 기온 데이터와 사용자 기분 데이터로 AI가 상황에 가장 적합한 음식을 추천합니다.   

### 핵심
사용자의 음식 선택에 있어서 `결정하는 시간을 단축`하고, `상황에 맞는 가장 좋은 음식`을 추천해 불필요한 시간 소요를 줄이는 것입니다.

### ✨주요기능
* **실시간 기온 분석:** 사용자의 현재 위치 기반으로 실시간 기온과 날씨 정보를 수집
* **기온 맞춤 AI 음식 추천:** 현재 위치의 기온에 맞춰 최적의 음식 추천
* **메뉴 상세:** 추천된 음식의 이미지, 칼로리 등 상세 정보 전달
* **(선택)배달 앱 연동:** 추천된 음식을 선택할 경우, 해당 음식을 배달 앱에서 바로 주문할 수 있도록 연동 실행(확인 필요)

### ⚙️사용 기술 스택
지금까지 주로 `React`, `TypeScript`, `Next.js`의 자바스크립트 기반의 프레임워크를 사용했습니다. 그러나, 이번 미션에서는 이를 제외하고 처음 사용해보거나 1~2번의 사용 외에 거의 경험해보지 못한 기술을 선택할 계획입니다.   

또한, API 요청 등 새로운 언어의 간단한 백엔드 구현을 새로운 도전으로 목표합니다.   

* **프론트엔드:** Vue
* **백엔드:** Python, Rust(포기)
* **디자인:** Figma
* **API:** 공공데이터 포탈 API

### 🚀예상 동작
```
사용자 ─▶ 프론트엔드 ─▶ 백엔드 서버 ─▶ 날씨 API
   │             │              │
   │             │◀─────────────┘ (현재 위치의 날씨, 온도, 체감온도, 미세먼지)
   │             │
   │             ├──▶ AI 추천 API (기분 + 날씨 데이터 전송)
   │             │
   │             ◀── 음식 추천 결과 수신 (예: "따뜻한 라멘")
   │
   ◀──────────── 음식 추천 결과 반환
```

**미니 게임 - 로또**

```
   │             │◀─────────────┘ (예: [5, 12, 23, 34, 39, 41])

```

### 미니게임
2~3주 차 미션인 `자동차 경주`와 `로또` 게임을 터미널이 아닌 실물로 구현해 사용자가 미니게임 형식으로 사용자가 간단하게 즐길 수 있도록 구현할 계획입니다.   

두 개의 게임 중 최소 한 가지 이상 구현하는 것이 목표입니다.   

**예상 동작**
```
사용자 ─▶ 프론트엔드 ─▶ 백엔드 서버 ─▶ 랜덤 번호 생성기
   │             │              │
   │             │◀─────────────┘ (예: [5, 12, 23, 34, 39, 41])
   │             │
   │             ├──▶ DB (사용자 미션 진행 상태 저장)
   │             │
   │             ◀── 결과 반환 (보상, 성공/실패 여부)
   │
   ◀──────────── 결과 표시 (예: "축하합니다! 3개 일치 🎉")
```

### 구현 후기
- **Rust 활용 포기:** Rust 특성 상 대규모 프로젝트에 어울린다는 것을 확인하였고, 로또 게임에서 활용할 경우, 단순히 난수 생성을 하거나 API로 로또 번호를 가져오려고 하는 것만으로는 Rust를 제대로 활용했다고 보기 어려워 포기하였습니다.
- **React와 Vue의 유사성:** 이번에 `Vue`를 사용하면서 느낀 것은 문법만 다를 뿐, 동작 과정은 `React`와 굉장히 유사하다는 점이었습니다. 쉽게 생각하면, 애플과 삼성처럼 같은 핸드폰이지만 동작은 비슷하지만 사용 방법이 다른 제품으로 선호도에 따라 선택하면 좋을 것 같습니다.
- **FastAPI의 편리함:** 개인 프로젝트 등을 구현하다보면 자주 접하는 문제가 `CORS` 문제였습니다. 그러나, 이번에 `Python`의 `FastAPI`를 사용하면서 `CORSMiddleware`를 추가해 쉽게 `CORS` 문제를 해결하는 것을 보며 다른 언어보다 편리성을 제공하는 언어라는 것을 느꼈습니다. 이번에 처음 사용해봤기 때문에 제대로 파악하지 못했지만, 이번 기회에 천천히 공부해보며 이해도를 높이고자 합니다.
- **공공데이터 포탈 API:** 알고 있었지만 사용해보지 않았던 `공공데이터 포탈 API`를 사용하면서 필요한 데이터를 하나의 페이지에서 쉽게 얻을 수 있다는 장점을 확인하였습니다. 한 번의 활용신청으로 위치, 기온 등 원하는 데이터를 얻을 수 있었고, 다음에도 적극 활용할 것 같습니다.
